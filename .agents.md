# Frontend Landing WebUI Component Rules (`gii-cshelpdesk-landing`)

This file contains the specialized design standards, layouts, and coding rules for the **Landing Website** component.

---

## 🏗️ Technology Stack
* **Language**: HTML5, Vanilla client-side JavaScript.
* **Styling**: Vanilla CSS3 (Custom Properties & Design Tokens).

---

## 🎯 Mandatory Design & Coding Standards

### 1. Premium Visual Aesthetics
* Use vibrant colors, sleek dark modes, subtle micro-animations, and glassmorphism.
* Strictly follow the predefined CSS color tokens:
  * `--color-primary: #008060` (Shopify Green)
  * `--color-secondary: #5c6ac4` (Shopify Indigo)
  * `--radius-lg: 12px` (Modern border radius)
* Use modern typography (Google Fonts 'Inter' and 'Outfit') instead of standard browser defaults.

### 2. SEO Rigorous Compliance
* **Single `<h1>` Constraint**: Every HTML page **MUST** contain exactly **one** `<h1>` tag, placed in the main hero or header section. All secondary titles must use `<h2>`, `<h3>` etc. in hierarchical order.
* **Unique Test IDs**: All interactive elements (form fields, buttons, inputs) **MUST** have unique, distinct `id` attributes to facilitate automated browser testing.
* *Example*:
  ```html
  <input type="email" id="reporterEmail" name="email" required />
  ```

### 3. Zero-Storage Privacy Policy Highlight
* Any page that collects customer data (like support forms, registration pages, bug reports) **MUST** feature a prominent, secure-looking callout card reassuring users of our **Zero-Storage Privacy Policy**.
* State clearly that the application processes emails/data purely in-memory and never logs or persists PII (Personally Identifiable Information) on our servers, ensuring absolute CCPA/GDPR compliance.

### 4. Centralized Site Navigation
* DO NOT hardcode global navigation headers or footers across individual page footers.
* All global layout headers and footers are dynamically generated by `js/common.js` inside the `renderChrome()` method.
* When adding a new landing page (e.g. `support.html`, `issue.html`), register the link inside the footer arrays in `common.js` so it automatically synchronizes across all pages of the site.

---

## 🧪 Verification Checklist
* Load pages in local live server, verify on Chrome DevTools under various mobile, tablet, and desktop viewports.
* Check accessibility (contrast ratio, descriptive labels) on all forms using audits to verify excellent UX.
* Run a quick grep search on new page structures to assert that the "exactly one `<h1>`" rule is 100% compliant.
